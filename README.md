
# BitShifter

 Программа разработаная для автоматического применения XOR маски к файлам.

 

## Features

- Параллельное выполнение всех созданных исполнителей
- Добавление нескольких файлов через Input file mask
- Динамическое отображение текущего прогресса
- Кроссплатформенность


## Input file mask

Для добавления нескольких файлов могут использоваться несколько путейб разделённых ";", либо wildcard symbol "*"
При поиске файлов WS будет заменён на все возможные папки (в случае если он  не в конце маски поиска), или на все файлы с расширением после WS (в случае, если он последний в маске поиска).

#### Примеры:
InputDir\
├── Mon\
│   ├── 12_00.txt\
│   ├── 13_45.txt\
│   └── 16_18.txt\
├── Tue\
│   ├── 12_00.txt\
│   ├── 13_45.txt\
│   └── 16_18.txt\
├── Wed\
│   ├── 12_00.txt\
│   ├── 13_45.txt\
│   └── 16_18.txt\
└── Thu\
│   ├── 12_00.txt\
│   ├── 13_45.txt\
│   └── 16_18.txt\
...

- Маска: "InputDir/Wed/13_45.txt"
    - InputDir/Wed/13_45.txt

- Маска: "InputDir/*/13_45.txt"
    - InputDir/Mon/13_45.txt
    - InputDir/Tue/13_45.txt
    - InputDir/Wed/13_45.txt
    - InputDir/Thu/13_45.txt

- Маска: "InputDir/Mon/*.txt"
    - InputDir/Mon/12_00.txt
    - InputDir/Mon/13_45.txt
    - InputDir/Mon/16_18.txt

- Маска: "InputDir/* / *.txt"
    - Все файлы с расширением .txt


    
## Known limitations

### Input file mask

В данный момент * выполняет роль WildCard-а только на текущем уровне директори, хотя общепринято:

- "*" - Заменяется на любое количество символов

- "?" - Заменяется на один символов

Однако текущий вариант, как мне кажется, более удобным.

### Сохранение текущих исполнителей в файл на выходе

Нужно добавить функционал, позволяющий экспортировать и импортировать ACD::::TaskExecutor::TaskConfig-а в и из строк, чтоб их можно было сохранить в файл.

### Названия Stop/Abort

Далеко не лучший выбр имени не описывающее точно привязаные функции:

- Stop остановить текущий исполнитель после того как все файлы будут обработаны

- Abort остановить текущий исполнитель после того как будет обработан текущий файл
